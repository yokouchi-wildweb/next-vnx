# プロジェクト固有ルール

このディレクトリには、フォーク先のプロジェクト固有の指示を配置します。

## 使い方

1. このディレクトリに `.md` ファイルを追加すると、自動的に読み込まれます
2. ベースの `CLAUDE.md` は変更せず、追加ルールのみをここに記述します

## 例

```
.claude/rules/
├── README.md           # このファイル
├── project-config.md   # プロジェクト固有の設定・ルール
├── api-rules.md        # API固有のルール
└── domain/             # ドメイン別のルール（サブディレクトリも可）
    └── users.md
```

## 記述例（project-config.md）

```markdown
# プロジェクト固有ルール

## 概要
このプロジェクトは〇〇のためのアプリケーションです。

## 追加の技術スタック
- 外部API: Stripe（決済）
- メール: SendGrid

## 固有のルール
- 決済処理は必ず `src/features/payment/` 配下で行う
- メール送信は `src/lib/email/` のラッパーを使用する
```
